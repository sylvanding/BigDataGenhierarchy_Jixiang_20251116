# 背景介绍
大数据除了大容量（ Volume）和变化快（ Velocity）外，还有一个相对研究较少的重要特性或者面临的挑战就
是多样性（ Variety），一般泛指类型和来源的广泛。处理种类繁多的数据一般可以有两类方法：

- 专用方法，它为每种数据类型量身定做处理系统，性能好但代价高；
- 通用方法，通过构建通用的系统处理多种类型数据。

值得肯定的是，如果性能满足应用需求，通用方法具有很高的性价比，因而其往往成为应用软件开发的首选。构建通
用系统一般是先找出可以涵盖多种数据类型的统一抽象类型，然后根据统一类型的特点构建系统，这样的系统就可以
适用于统一类型所涵盖的多种数据类型了。

UMAD(Universal Management and Analysis of Data)是由深圳大学[毛睿教授][1]主持，致力于实现通
用的大数据管理分析系统的项目。该项目利用度量空间的高度抽象性的特点，将各种各样的数据类型抽象到度量空间
之中，而所有的索引算法都针对度量空间这一数据结构进行设计，以此达到算法通用的目的。

各种各样的数据如果想要抽象到度量空间之中，只需要用户定义一个距离函数即可。通过该距离函数，可以求出任意两个数据
点之间的距离。但是高度抽象性既是度量空间的优点，也是度量空间的缺点。由于度量空间的高度抽象性，用户仅仅
需要提供一个距离函数就可以将数据映射到度量空间之中，可与此同时，度量空间中的数据点能为检索提供的信息也
过于少。基于此，研究人员提出了支撑点空间模型，该模型通过在度量空间之中按照一定规则选择一定数目的支撑点，
将数据映射到支撑点空间之后，给数据加上了坐标，使得后续的众多数学工具可以使用。在支撑点空间处理完数据之后，
再映射回原本的度量空间，以此达到算法通用的目的。

# 项目介绍
本项目主要基于jdk12.0.2实现，同时也使用到了Python和C++。为了更好的支持中文，全部编码统一采用UTF-8。项目目前支持DNA、RNA、Peptide、Spectra、
Image和Vector数据库的建立和相似性搜索,以及对它们进行聚类分类分析，异常点检测分析等。项目所有的实验数据集在该项目`data`
目录下均可以找到，关于更详细的数据集说明，请参考☛[wiki文档数据集][4]。


# 包功能简短说明

| ---------- | ------------------------------------------------------------ |
| app        | 前端接口。快速构建的入口，封装了一些用户常用的情景的快速实现。 |

# 项目功能介绍
本项目支持度量空间索引的构建、查询，以及使用度量空间进行聚类、分类和异常点检测分析等。在索引部分，本项目支持VP、GH、KMP等索引树的构建，同时实现了常用的支撑点选择算法，如FFT、CENTER、RANDOM等；常用
的数据划分算法，例如Balanced、ClusteringKMeans、CGHT和GHT等。  
数据分析部分，项目实现了DBSCAN、BIRCH、KMeans聚类算法，KNN分类算法。关于异常点检测部分，考虑到检测数据集往往较大，为了兼顾运行效率，该部分是使用C++实现。
